# 图片上传功能实现

## 功能概述
为Rust Agent添加图片上传功能，使其能够将图片发送给支持视觉的LLM模型（如GPT-4V、Claude 3.5 Vision等）。

## 当前限制
1. ContentBlock枚举只有Text、ToolUse、ToolResult、Thinking类型
2. LLM客户端只处理文本消息
3. 没有图片处理工具
4. 模型配置不支持vision模型

## 实现步骤

### 1. 扩展消息结构
在 `src/conversation.rs` 中修改 `ContentBlock` 枚举，添加 `Image` 类型：
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum ContentBlock {
    #[serde(rename = "text")]
    Text { text: String },
    
    #[serde(rename = "image")]
    Image {
        source: ImageSource,
        #[serde(skip_serializing_if = "Option::is_none")]
        mime_type: Option<String>,
    },
    
    // ... 其他类型
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum ImageSource {
    #[serde(rename = "base64")]
    Base64 {
        media_type: String,
        data: String,
    },
}
```

### 2. 创建图片上传工具
创建 `src/tools/upload_image.rs`：
- 支持读取图片文件
- 将图片转换为base64编码
- 支持常见的图片格式（PNG、JPEG、GIF、WebP）
- 自动检测MIME类型

### 3. 更新LLM客户端
#### OpenAI客户端 (`src/llm/openai.rs`)：
- 支持OpenAI的图片消息格式
- 将Image ContentBlock转换为OpenAI API格式
- 处理base64编码的图片数据

#### Anthropic客户端 (`src/llm/anthropic.rs`)：
- 支持Claude的图片消息格式
- 将Image ContentBlock转换为Anthropic API格式
- 处理base64编码的图片数据

### 4. 添加模型配置
在 `docker/config/models.toml` 中添加vision模型配置：
```toml
[models.gpt4v]
provider = "openai"
base_url = "https://api.openai.com/v1"
model = "gpt-4-vision-preview"
max_tokens = 4096

[models.claude3_vision]
provider = "anthropic"
base_url = "https://api.anthropic.com"
model = "claude-3-5-sonnet-20241022"
max_tokens = 4096
```

### 5. 更新工具注册
在 `src/tools/mod.rs` 中注册新的图片上传工具。

## 使用示例
用户可以通过以下方式使用图片上传功能：
1. 使用 `upload_image` 工具上传图片
2. 图片会被转换为base64并添加到对话历史
3. LLM客户端会自动处理图片消息格式
4. 支持多模态对话（文本+图片）

## 注意事项
1. 图片大小限制：API通常有大小限制（如OpenAI限制为20MB）
2. 支持的格式：PNG、JPEG、GIF、WebP
3. 性能考虑：base64编码会增加约33%的数据量
4. 错误处理：处理无效图片文件、不支持的格式等
